## 新增功能

### 1. 🔴 检查点系统（Dim 4: +3.0）

在 AI Agent 执行关键操作前插入 3 处显式闸门，防止自主失控：

| 位置 | 类型 | 拦截内容 |
|------|------|---------|
| Step 0 — 生成 `_context.md` 前 | 🔴 CHECKPOINT | 确认 PDF 路径正确、文件未加密/未损坏 |
| Step 0.5 Phase C — 路由计划完成后 | 🛑 STOP | 向用户展示文档路由计划摘要（涉及哪些 PDF、查什么、排除理由），确认后继续 |
| Step 2B Phase 1 — 提取前 | 🔴 CHECKPOINT | 确认页码范围覆盖问题、控制在 20 页以内（避免上下文溢出） |

### 2. 🚫 反例与黑名单（Dim 9: +3.0）

新增独立章节，12 条禁止行为分 3 类，每条含「禁止行为 → 原因 → 正确做法」三联结构：

| 类别 | 条数 | 示例 |
|------|:--:|------|
| **流程安全** | 5 | 禁止跳过 index 直接 Glob context、禁止单次提取 >20 页、禁止无 TOC 盲目搜索 |
| **输出质量** | 4 | 禁止无 `[来源:]` 引用给出参数、禁止扫描件文本提取后下结论、禁止忽略 multimodal 标记 |
| **资源管理** | 3 | 禁止静默覆盖 `_context.md`、禁止残留 temp_images、禁止未验证 venv 调用脚本 |

### 3. 失败模式与异常处理（Dim 3: +6.0）

新增 16 条系统化三段式 fallback（触发条件 → 一线修复 → 仍失败兜底），覆盖全部关键路径：

| 模块 | 条数 | 覆盖场景 |
|------|:--:|---------|
| 环境与依赖 | 4 (E1-E4) | venv 缺失、pip 失败、PDF 缺失、PDF 加密 |
| 上下文缓存 | 2 (C1-C2) | `generate_context.py` 返回空 JSON、TOC 层级为 0 |
| TOC 解析 | 2 (T1-T2) | `parse_toc.py` 返回空、大 PDF 崩溃超时 |
| 页面提取 | 2 (P1-P2) | 扫描版乱码、超时/内存溢出 |
| 表格提取 | 2 (B1-B2) | 无表格检测、解析错位 |
| 图像导出 | 2 (I1-I2) | 空白图片、超时/磁盘不足 |
| 跨文档路由 | 2 (R1-R2) | 无 index 无 context、路由全 Low |

### 4. ⚡ 快速路径豁免（Dim 8: 延迟优化）

在 Step 0.5 跨文档预分析中新增快速路径豁免规则：

> 同时满足以下条件时，**跳过 Step 0.5 全部 Phase**，直接进入 Step 1：
> - 用户明确指定了单个 PDF 文件路径
> - 问题答案仅依赖这一个 PDF
> - 问题不涉及跨文档交叉引用

**影响**: 单文档简单查询（如"这个芯片的 VDD 是多少"）的流程步骤从 ~8 步降至 ~3 步，大幅降低 Agent 执行延迟。

## Bug 修复

### 资源路径一致性（Dim 6: +1.6）

**问题**: SKILL.md 中 8 处脚本路径引用 `skills/datasheet_reader/scripts/`，与实际 skill 安装目录名 `datasheet` 不一致，导致脚本路径不可达。

**修复**: 全部修正为 `skills/datasheet/scripts/`，并验证 6 个脚本文件实际存在于目标路径。

| 变更文件 | 变更数量 |
|---------|:------:|
| `skills/datasheet/SKILL.md` | 8 处路径引用 |
| Prerequisites 命令 | 1 处 |
| Step 0 命令 | 1 处 |
| Step 1 命令 | 1 处 |
| Step 2A 命令 | 1 处 |
| Step 2B 命令 | 3 处 |
| Script Reference 描述 | 1 处 |

### Interface Spec 契约对齐（Dim 5: +1.7）

**问题**: `query` action 的说明为通用描述，未反映实际执行时 Step 0.5 跨文档路由的开销差异。

**修复**: Actions 表中 `query` 和 `summarize` 的说明已明确标注快速路径 vs 完整路由的触发条件。
